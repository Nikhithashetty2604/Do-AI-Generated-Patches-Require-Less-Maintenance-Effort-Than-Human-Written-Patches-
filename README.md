Do AI-Generated Patches Require Less Maintenance Effort Than Human-Written Patches?

This repository contains the code, scripts, and experimental results used to compare the maintenance effort of AI-generated patches and human-written patches.
The study evaluates patch-level characteristics using bugs from the SWE-bench dataset and measures metrics that approximate maintenance effort.

The goal of the project is to investigate whether patches generated by large language models require less maintenance effort compared to patches written by developers.

Dataset

The bugs used in this study are selected from the SWE-bench dataset:

princeton-nlp/SWE-bench

The selected bug instances come from the following open-source repositories:

django/django

pytest-dev/pytest

psf/requests

These projects were chosen because they are widely used Python projects with well-documented bug fixes.

Human Patch Analysis

Human-written patches are analyzed using the script:

scripts/analyse.py

This script processes the human patches and extracts maintenance-related metrics such as:

patch size

number of modified lines

number of affected files

other patch-level characteristics

Running the Human Patch Analysis
python scripts/analyse.py
Output

The computed metrics are stored in:

outputs/Human_bug_metrics

Each bug processed by the script produces a corresponding entry containing the computed patch metrics.

AI Patch Generation

AI-generated patches are produced using the script:

scripts/Generate_AI_Patch.py

This script generates patches by querying the CodeLlama 7B Instruct model through a Hugging Face inference endpoint.

The script performs the following steps:

Load selected bug instances from SWE-bench

Extract the buggy file and code context

Build a prompt describing the bug

Query the CodeLlama model

Generate a unified git diff patch

Validate and store the generated patch

Important Requirement

Running the AI patch generation script requires access to a Hugging Face Inference Endpoint hosting the CodeLlama model.

Model used:

codellama/CodeLlama-7b-Instruct-hf

Creating and running this endpoint may require a paid Hugging Face inference endpoint, depending on the compute configuration.

Environment Setup

Before running the script, set the required environment variables.

Windows CMD
set HF_TOKEN=your_huggingface_token
set CODELLAMA_ENDPOINT=https://your-endpoint-url

The token and endpoint are required to authenticate and send requests to the model.

Running AI Patch Generation
python scripts/Generate_AI_Patch.py

The script will automatically:

load the selected SWE-bench bugs

extract the relevant buggy code

query the CodeLlama model

generate patches

validate them

Output

Generated AI patches are saved in:

outputs/Codellama_AllPatches

Each bug instance produces a .patch file containing:

the generated patch

a validation status indicating whether the patch follows the expected format.

Reproducing the Experiment

To reproduce the experiment:

Clone this repository

git clone https://github.com/Nikhithashetty2604/Do-AI-Generated-Patches-Require-Less-Maintenance-Effort-Than-Human-Written-Patches-.git

Install required Python dependencies

pip install datasets requests

Clone the required repositories into the repos/ directory

Example:

git clone https://github.com/django/django repos/django
git clone https://github.com/pytest-dev/pytest repos/pytest
git clone https://github.com/psf/requests repos/requests

Run the analysis scripts.

Notes

The repos/ directory contains cloned repositories that are used to locate buggy files and compute patch metrics.
These repositories are not included in the GitHub repository to keep the project lightweight.

Generated patches and experiment outputs are included so that the results of the study can be inspected without running the model.
