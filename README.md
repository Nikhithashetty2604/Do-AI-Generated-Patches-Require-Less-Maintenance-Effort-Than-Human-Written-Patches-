# Do AI-Generated Patches Require Less Maintenance Effort Than Human-Written Patches?

This repository contains the code, scripts, dataset artifacts, and experimental results used to compare the maintenance effort of AI-generated patches and human-written patches.

The study evaluates patch-level characteristics using bug instances from the SWE-bench dataset and measures metrics that approximate maintenance effort in software maintenance tasks.

The objective of the project is to investigate whether patches generated by large language models (LLMs) require less maintenance effort compared to patches written by human developers.

# Dataset

This study uses bug instances from the SWE-bench dataset:

princeton-nlp/SWE-bench

The selected bugs come from the following open-source Python repositories:

- django/django
- pytest-dev/pytest
- psf/requests

These repositories were selected because they are widely used, actively maintained, and contain well-documented bug fixes.

A total of 16 bug instances are used in the experiment.

Metadata such as:

- Bug ID
- Repository
- Bug description
- Base commit

is stored in:

data/swebench_bugs/bug_details.csv

---

# Human Patch Collection

Human-written patches are extracted from SWE-bench using:

scripts/Collect_Human_Patch.py

This script performs the following steps:

1. Load the SWE-bench dataset
2. Filter the selected bug instances
3. Extract the developer-written patch
4. Save each patch as a .patch file

Human patches are stored in:

data/patches/human

---

# AI Patch Generation

AI-generated patches are produced using:

scripts/Generate_AI_Patch.py

This script generates patches by querying the CodeLlama 7B Instruct model using a Hugging Face inference endpoint.

## Patch Generation Process

The script performs the following steps:

1. Load the selected SWE-bench bug instances
2. Extract the buggy code context
3. Build a prompt describing the bug
4. Query the CodeLlama model
5. Generate a unified git diff patch
6. Validate and store the generated patch

Generated patches are saved in:

data/patches/ai

---

# Model Used

AI patches are generated using the following model:

codellama/CodeLlama-7b-Instruct-hf

The model is accessed through a Hugging Face inference endpoint.

Running the script requires authentication using a Hugging Face token and endpoint URL.

# Hugging Face Configuration

Before generating AI patches, set the required environment variables.

## Windows CMD

set HF_TOKEN=your_huggingface_token  
set CODELLAMA_ENDPOINT=https://your-endpoint-url

These values are used to authenticate requests sent to the model endpoint.

# Metrics and Results

Patch-level metrics that approximate maintenance effort are computed and stored in:

outputs/Ai_Patch_metrics.csv  
outputs/Human_bug_metrics.csv  

These metrics include characteristics such as:

- patch size
- number of modified lines
- number of affected files
- other patch-level attributes related to maintenance effort

The results are used to compare **AI-generated patches and human-written patches**.

# Research Context

This project explores whether AI-generated bug fixes require less maintenance effort compared to human-written patches.

The study contributes to understanding how large language models can assist software maintenance tasks and whether AI-generated patches can match the maintainability of developer-written fixes.
